# LAB2 博饼实验报告

## 一、实验目的
1. 熟悉 Java 控制台程序的基本开发流程。
2. 掌握随机数生成、数组统计、条件判断、循环控制等基础语法。
3. 学会根据题目要求设计博饼奖项判定逻辑。
4. 理解模块化编程思想，将掷骰、判奖、奖池管理、玩家记录等功能分开实现。
5. 提高程序调试和测试能力，能够使用测试数据验证程序结果。

## 二、实验内容
1. 编写一个 Java 控制台程序，实现“博饼”题目要求。
2. 随机生成 6 个骰子点数，并根据规则判断奖项。
3. 实现基本题要求，支持一秀、二举、三红、四进、对堂、状元档等奖项判定。
4. 实现扩展题要求，支持 6 到 10 名玩家轮流掷骰。
5. 设计奖池库存，命中奖项后按库存发奖，库存为空时提示“命中但奖项已发完，不得奖”。
6. 在游戏结束后输出每位玩家获得的奖项明细，以及每类奖项的发放数量和剩余数量。

## 三、算法流程
本程序整体流程如下：

1. 程序启动后显示菜单，可选择单次判奖、多人模式或退出。
2. 若选择单次判奖，则随机生成 6 个骰子点数。
3. 统计 1 到 6 各点数出现次数。
4. 按题目要求的优先级依次判奖：
   - 先判断所有状元档
   - 再判断对堂
   - 再判断三红
   - 再判断四进，并判断是否附带一秀或二举
   - 再判断二举
   - 再判断一秀
   - 否则判定为无奖
5. 若选择多人模式，则先输入玩家数，范围为 6 到 10。
6. 所有玩家按顺序轮流掷骰并判奖，若库存足够则发奖，否则提示该奖项已发完。
7. 当所有奖项库存都为 0 时，结束多人模式。
8. 最后输出每位玩家获奖明细，以及各类奖项统计结果。

## 四、核心功能实现
说明：代码文件已单独提供，这里只说明关键实现点。

1. 使用 `Random` 生成 6 个 `1~6` 之间的随机骰子点数。
2. 使用数组统计各点数出现次数，便于判断各种奖项。
3. 将判奖逻辑封装在独立类中，严格按题目规定的优先级判定。
4. 状元档单独判断，命中后不再继续叠加低阶奖项。
5. 对“四进带一秀”和“四进带二举”进行了组合奖处理。
6. 使用奖池类保存库存，命中奖项后扣减库存；若库存已空，则提示该奖项已发完。
7. 使用玩家记录类保存每位玩家的获奖明细，便于最后统一输出。
8. 采用菜单模式组织程序，既可以演示基本题，也可以演示扩展题。

### 类划分说明
- `Main`：程序入口，负责菜单显示、用户输入、单次判奖和多人模式流程。
- `DiceRoller`：负责随机生成 6 个骰子点数。
- `BoBingEvaluator`：负责按照题目规则判断奖项。
- `RollResult`：保存一次掷骰的判奖结果。
- `PrizePool`：负责奖池库存管理和实际发奖。
- `AwardOutcome`：保存本轮实际发出的奖项和库存已空的奖项。
- `PlayerRecord`：记录每位玩家获得的奖项明细。
- `PrizeType`：表示奖项类别。
- `ChampionType`：表示状元档细分类型。
- `TestCases`：提供固定测试数据，验证程序正确性。

## 七、出现的问题及解决方法
1. 问题：状元档和普通奖项之间可能发生冲突。  
   解决方法：严格按题目要求，先判断所有状元档，命中后立即返回，不再继续判断低阶奖项。

2. 问题：五子容易被误判为四进加一秀。  
   解决方法：在判断四进之前，先判断所有状元档中的五子，从而避免误拆。

3. 问题：四进可能同时附带一秀或二举。  
   解决方法：先判断是否存在非 4 的四个相同点数，再根据剩余 4 的数量决定附带一秀还是二举。

4. 问题：多人模式中库存管理较复杂。  
   解决方法：单独设计奖池类，对每一种奖项分别记录剩余数量和已发数量，发奖时逐项检查库存。

5. 问题：用户输入玩家人数时可能输入非法内容。  
   解决方法：加入输入校验，若输入不是整数或不在 6 到 10 范围内，则提示重新输入。

## 八、实验心得
通过本次实验，我进一步掌握了 Java 控制台程序的基本编写方法，并且练习了数组统计、条件判断、循环控制、面向对象封装等知识。在实现过程中，我体会到程序设计不能只关注“能运行”，还要特别注意题目规则和判定顺序，否则很容易出现逻辑错误。

本次实验中最关键的部分是奖项优先级和组合奖处理。通过将判奖逻辑、奖池管理、玩家记录分别封装到不同类中，程序结构更加清晰，也更容易调试和扩展。通过固定测试数据进行验证，我也认识到测试在程序开发中的重要性。今后在编写程序时，我会更加重视模块划分、边界情况处理和测试验证。

## 运行方法
### Eclipse
1. 打开 Eclipse。
2. 选择 `File -> Open Projects from File System...`。
3. 导入 `LAB2` 目录。
4. 运行 `src/Main.java` 开始程序。
5. 运行 `src/TestCases.java` 查看固定测试样例。

### 命令行
在 `LAB2` 目录下执行：

```bash
javac -d bin src/*.java
java -cp bin Main
java -cp bin TestCases
```
