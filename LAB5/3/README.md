# LAB5 第3题：painter 程序中的多态设计说明

本题分析的对象是仓库中 `LAB4/4/target/classes/RandomShapeDrawerApp*.class` 对应的随机图形绘制程序。虽然当前仓库没有保留它的 `.java` 源文件，但从已编译的类可以看出，这个程序在“根据图形类型组织行为”这一点上体现了多态思想。

## 1. 图形类别的统一抽象

程序把不同图形统一抽象为 `ShapeType`。当前可见的图形类别有三种：

- 直线 `LINE`
- 矩形 `RECTANGLE`
- 椭圆 `ELLIPSE`

这些图形虽然绘制方式不同、参数含义不同，但在程序里都被当作“当前待绘制图形类型”来处理。也就是说，主流程不需要为每种图形单独写一套完全独立的界面逻辑，而是围绕统一的图形类型抽象展开。

## 2. 统一接口思想

程序对不同图形采用同一套处理流程：

1. 随机生成当前图形类型。
2. 根据图形类型给出参数名称和取值范围。
3. 读取并校验用户输入。
4. 按当前图形类型执行对应的绘制逻辑。

这说明程序虽然面对的是三种不同图形，但外部流程是一致的。也就是“同一入口、同一处理框架、不同类型执行不同具体行为”。这种设计体现了多态的核心思想。

## 3. 运行时绑定体现

程序运行时不会在一开始就固定只处理某一种图形，而是先随机得到当前图形类型，再决定后续行为。例如：

- 如果当前类型是直线，就要求输入起点和终点坐标；
- 如果当前类型是矩形，就要求输入左上角坐标、宽度和高度；
- 如果当前类型是椭圆，就要求输入中心坐标和横纵半径。

因此，程序在运行过程中会根据“当前对象/当前类型”的不同，表现出不同的参数说明、校验规则和绘制结果。这种“运行时决定具体行为”的特征，就是多态思想在程序中的体现。

## 4. 可扩展性优势

这种设计方式的一个重要优点是便于扩展。若后续要增加新图形，例如圆、三角形或多边形，只需要补充新的图形类型，并为它定义对应的参数信息和绘制规则，主流程仍然可以复用原来的“随机生成类型 -> 输入参数 -> 校验 -> 绘制”框架，不需要整体推翻程序结构。

因此，这个 painter 程序的多态设计价值主要体现在两点：

- 降低主流程对具体图形实现的耦合；
- 让程序更容易维护和扩展。

## 5. 总结

`RandomShapeDrawerApp` 并不是简单地把所有图形逻辑硬编码在一起，而是先对图形类别进行统一抽象，再在运行时根据具体图形类型切换行为。不同图形共享同一套外部处理流程，却能够表现出各自不同的参数和绘制效果，这正是多态设计在该程序中的核心体现。
